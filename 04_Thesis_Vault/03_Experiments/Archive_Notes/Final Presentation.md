# Background
The Challenge of Legacy ECGs:  
	The Problem: Billions of ECGs exist only as paper records or images (PDF/JPG), making them unusable for modern AI/ML diagnostics. 
	ML models require clean time-series data, not images. 
	Digitization - converting these images back into clean, 1-D time-series data.
	Reality: Existing digitization pipelines (e.g., ECG-Image-Kit) introduce artifacts
	Impact: Poor digitization → wrong morphology → wrong diagnosis. 
	Goal: Improve signal quality after digitization.

ECG Image Digitization Pipeline [BASELINE]
	A[Input: Scanned ECG Image] --> B[Step 1: Pre-processing]
	B --> B1[Lead Marker Detection using "MarkerFInder"]
	B --> B2[Grid Removal (Color Images)]	
	B --> C[Step 2: Signal Extraction]
	C --> C1[⟐ Image Type Check\nHeight & color channels]
	C1 --> C2[Column Sweep (Color) **Signal = (top + bottom) / 2**]
	C2 --> C3[Ensemble Avg : Column Sweep + NN predictions]
	C1 --> C4[Neural Network (B&W) Dense model]
	C --> C6[Scale to mV\nPixel ÷ 80 px/mV]	
	C --> D[Step 3: Post-processing]	
	D --> E[ Output: 12-Lead Time Series (mV)]

# Problem Statement
Even after full digitization, the waveform still contains noise.
Hypothesis: We need a post-digitization refinement stage.
The Idea: Use a Conditional Diffusion Model for post processing.

Problem Statement: Can diffusion models restore clinically meaningful ECG morphology from imperfect digitized signals where conventional denoising fails?

# what we did after that
 Central Finding of This Semester

**The model was never the problem. The data was.**

We spent this semester diagnosing the real failures and found **four major bugs in the extraction pipeline** — none of which were visible in the results because the model was overfitting noise.

Once all four bugs were fixed, the same model architecture achieved **+15.7% MSE improvement** on real data.

# The Four Extraction Bugs

| Bug                      | What was wrong                                                                                                   | What we fixed                                |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| **1. Signal length**     | TARGET=2500, but real signal is 1250 samples. Every other point was a duplicate                                  | TARGET=1250                                  |
| **2. Normalization**     | Clean and noisy both scaled to [-1,1] independently, which erased the Amplitude errors. DM had nothing to learn. | Noisy.npy scaled using **clean.npy's range** |
| **3. Sample count**      | Only 10 ECG images → 120 samples. Far too few for a DM to learn real noise                                       | Extract until 500 good pairs                 |
| **4. Digitizer quality** | Used color sweep only (weaker)                                                                                   | Used **ensemble** (sweep + NN)               |

# Model Architecture

**Conditional DDPM with 1D UNet**

- Input: noisy diffusion signal x_t + conditioning signal (digitized ECG)
- Output: predicted noise ε
- 7.27 million parameters


**Training:** 200 epochs, LR =0.0001, batch=32, train/test= 400 train / 100 val

**Inference:** Partial reverse diffusion t_start=30 (refinement, not full generation from scratch)

Full Generation (t_start = 200) • Starts from pure random noise • Runs all 200 reverse steps • Generates signal completely from scratch • Risk: hallucinating ECG morphology not present in the digitized input

Refinement (t_start = 30) • Starts from the digitized signal • Adds only a small amount of noise (15% of schedule) • Runs only 30 reverse steps • Makes targeted corrections: smooths staircase, reduces spikes, adjusts baseline

# Limitations
- V4, V5, V6 leads have only 7–10 training samples — results unreliable for those leads
- Integration gain (+0.68%) is modest due to normalization mismatch between training and inference
- Model trained on 500 samples from 69 ECG images — limited diversity of scan quality, paper types, printer artifacts
- All training data is 500Hz — untested on 250Hz or 1000Hz ECGs
- Needs evaluation on other digitizers

# Future work
**Scale the dataset** and **Retrain the model** - needs a lot of compute
**Evaluate on ECG-Image-Kit** — replace DnCNN with the DM inside ECG-Image-Kit's pipeline, compare directly
**Faster sampling** — replace DDPM with DDIM or DPM-Solver (same model, 10x faster inference, important for clinical deployment) - needs more compute

# Conclusion
Diffusion models are capable of effectively refining post-digitization ECG signals.
